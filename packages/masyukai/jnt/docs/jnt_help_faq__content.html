FAQ

How to deal with error message "code 145003030, headers signature failure"?
The headers signature is a signature generated by the combination of privatekey+bizContent. If it prompts that the header signature failed, you need to check whether the signature method is used correctly. You can go to the Help Center to view the signature rules and compare it with the results generated by the Signature Tool.
What is the corresponding status of each scanning node in the Tracking Info?
10: Pick Up
20: Departure
30: Arrival
94: Delivery
100: Delivery Signature
110: Problematic
172: Return
173: Return Delivery Signature
200: Collected
Where can you obtain the apiAccount, privateKey, customerCode, and password (plain text password) of the Production Environment?
Production Environment apiAccount and privateKey can be checked inside Console - Personal Center, supporting multiple application creation to receive multiple apiAccount; customerCode and plain-text password need to contact the DP to create the Open Platform Account and Password.
Already launched API, can be Joint-Debugging again?
In Console - Application Management, you can initiate Joint-Debugging for the API that has passed Joint-Debugging again, but will not change the API Status.
What's the function of User Profile's Application Management?
When there are multiple Business Models, one apiAccount may not be able to meet the needs, and multiple apiAccounts need to be added through application management.
